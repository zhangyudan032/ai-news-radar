# Advanced Source Free-Tier & Budget Plan — X API / AgentMail

Date: 2026-05-10
Scope: AI News Radar 2.0 advanced/private sources. Public default must still run without secrets, browser login, cookies, or paid usage.

## Executive conclusion

1. **X API should not be treated as a free daily source.** The current X API docs describe pay-per-usage pricing: read operations are charged per resource returned. For AI News Radar, direct X API reads must stay opt-in, secret-backed, and budget-capped.
2. **AgentMail has a useful free tier for personal newsletter intake.** The public pricing page lists Free as `$0/month`, `3 inboxes`, `3,000 emails/month`, `100 emails/day`, and `3 GB storage`. This is enough for a personal AI newsletter inbox if we only publish metadata.
3. **Implementation policy:** both sources are visible as advanced capabilities but default off. X API uses a once-daily run window plus `X_API_DAILY_POST_LIMIT`; AgentMail exports only sender domain, subject/preview snippets, timestamp, and attachment count. No body, raw `.eml`, full email address, token, or inbox secret is written to the repo.

## Official-source findings

### X API

Checked official docs:

- Pricing: `https://docs.x.com/x-api/getting-started/pricing.md`
- Rate limits: `https://docs.x.com/x-api/fundamentals/rate-limits.md`
- Recent search: `https://docs.x.com/x-api/posts/search/introduction.md`
- Usage endpoint: `https://docs.x.com/x-api/usage/introduction.md`

Key facts from the checked docs:

- X API pricing is currently described as **pay-per-usage** through purchased credits, not as a free read quota.
- Read operations are charged per resource returned. Relevant rows:
  - `Posts: Read`: `$0.005` per resource.
  - `User: Read`: `$0.010` per resource.
  - `List: Read`: `$0.005` per resource.
- Recent search endpoint:
  - `GET /2/tweets/search/recent`
  - Searches Posts from the last 7 days.
  - Listed as available to all developers.
  - Rate limit table shows `450/15min` per app, `300/15min` per user; result size `10 default`, `100 max`, query length `512`.
- Usage endpoint:
  - `GET /2/usage/tweets`
  - Tracks Post usage and daily breakdown.
  - Docs mention a pay-per-usage monthly cap of `2 million Post reads`.

Interpretation for AI News Radar:

- There is no safe “free daily X read budget” to spend automatically.
- The cheapest useful official route is one daily `recent search` request with `max_results=10`, estimated at `10 × $0.005 = $0.05` per run if 10 posts are returned.
- Account timelines and lists can be useful, but they still consume paid read resources and can import more social noise.

### AgentMail

Checked official docs:

- Pricing: `https://agentmail.to/pricing`
- Rate limits: `https://docs.agentmail.to/knowledge-base/rate-limits`
- List Messages API: `https://docs.agentmail.to/api-reference/inboxes/messages/list.mdx`
- Docs index: `https://docs.agentmail.to/llms.txt`

Key facts from the checked docs:

- Free tier:
  - `$0/month`
  - `3 inboxes`
  - `3,000 emails/month`
  - `100 emails/day`
  - `3 GB storage`
  - No custom domains on Free; custom domains start on paid tiers.
- API rate limits:
  - Endpoints are rate-limited per API key.
  - Over-limit response is `429 Too Many Requests` with `Retry-After`.
- List messages endpoint:
  - `GET https://api.agentmail.to/v0/inboxes/{inbox_id}/messages`
  - Supports query parameters including `limit`, `page_token`, `labels`, `before`, `after`, and `ascending`.

Interpretation for AI News Radar:

- AgentMail is suitable for personal newsletter intake under Free, as long as we do not create a high-volume outbound agent.
- For public site safety, only a metadata digest should be generated by default: sender domain, redacted subject, redacted preview, timestamp, attachment count.
- `data/email-digest.json` should not be committed unless `EMAIL_DIGEST_PUBLISH=1` is explicitly set.

## Daily budget rules

### Default public repo

| Source | Default | Network call | Cost/Quota impact | Public output |
| --- | --- | --- | --- | --- |
| OPML/RSS examples | Enabled only when maintainer passes OPML | Public RSS only | Free | Normal news items |
| X API | Off | None | `$0` | Capability status only |
| AgentMail | Off | None | `$0` | Capability status only |

### Personal opt-in profile

| Source | Suggested cap | Estimated daily spend / usage | Rationale |
| --- | ---: | ---: | --- |
| X API recent search | `X_API_MAX_RESULTS=10`, `X_API_DAILY_POST_LIMIT=10` | up to `$0.05/day` | Enough to demo official X search without quietly burning credits. For a builder-account demo, use `(from:karpathy OR from:swyx OR from:amasad OR from:sama OR from:garrytan) -is:retweet -is:reply`; see `docs/guides/x-api-demo-config.md`. |
| AgentMail inbox | `AGENTMAIL_LIMIT=25`, one inbox for newsletters | counts against `100 emails/day` / `3,000 emails/month` Free tier | Enough for daily AI newsletters and release emails. |

Notes:

- X API costs are based on resources returned, so a request returning fewer posts costs less than the max estimate.
- The workflow runs every 30 minutes. X API is additionally gated by `X_API_RUN_UTC_HOUR` and `X_API_RUN_UTC_MINUTE_MAX` so it does not run every cron tick.
- `X_API_FORCE_RUN=1` exists for local/manual testing only. Do not set it in scheduled GitHub Actions unless you knowingly accept repeated paid usage.

## Environment variables

Use `examples/advanced-sources.env.example` as the copyable template.

### AgentMail

- Secret: `AGENTMAIL_API_KEY`
- Secret/variable: `AGENTMAIL_INBOX_ID`
- Variable: `EMAIL_DIGEST_ENABLED=1`
- Variable: `AGENTMAIL_LIMIT=25`
- Optional variable: `AGENTMAIL_ALLOWED_SENDER_DOMAINS=alphasignal.ai` to keep a shared inbox scoped to one newsletter demo.
- Optional variable: `EMAIL_DIGEST_PUBLISH=1` only if publishing the redacted digest is intentional.

### X API

- Secret: `X_BEARER_TOKEN`
- Variable: `X_API_ENABLED=1`
- Variable: `X_API_QUERY=...`
- Variable: `X_API_MAX_RESULTS=10`
- Variable: `X_API_DAILY_POST_LIMIT=10`
- Variable: `X_API_RUN_UTC_HOUR=0`
- Variable: `X_API_RUN_UTC_MINUTE_MAX=10`
